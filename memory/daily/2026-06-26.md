**用户需求**:买卖信号→自动产生记录+提醒手动操作;记录带建议价 vs 实际价(系统 vs 手动);持续跟踪评价是否稳定盈利,据此判断是否接券商真自动买卖。

**6 问锁定设计**(详见 `docs/progress/2026-06-26-paper-trading-loop-design.md`):
1. 实际价归属 = **Trade 账本**(source_ref→Draft;manual→broker_api 无缝)
2. 持仓/盈亏 = **从 Trade 账本事件溯源推导**(推翻旧 Holding-only)
3. 卖出 = 4 信号(论点失效优先);建议卖价=风控类现价/止盈类公允×1.3
4. 回填 = UI"确认成交"弹窗;7天过期=cancelled;实际可自由偏离建议
5. 评价 = 四层(组合/交易/双引擎归因/信号质量)+沪深300基准+夏普
6. CSV 真实持仓→开仓 Trade;一本账+归因分离(双引擎只算 source_ref 非空);提醒=in-app system_alert

**落地计划**:P0 闭环地基(position_service Trade 派生 + Draft 回填 + 论点失效卖出 + in-app 提醒)→ P1 评价系统 → P2 补全卖出触发(估值止盈/仓位超限/news·earnings 接线)→ P3 清理债。

**产出**:设计文档 + TaskCreate #1-8(含依赖关系)+ 更新 harness auto-memory(Holding-only 反转决议)。
