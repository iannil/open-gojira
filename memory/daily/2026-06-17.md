1. **逃生口是 must-have**: M2 in_circle filter 默认开启后,11 个 plan_runner 测试失败 (test stocks 默认 in_circle=False). 加 `Plan.disable_in_circle_filter` 后,用户在 plan 级有逃生口. 但测试 fixture 必须显式 `in_circle=True` 才能跑通 — 反映 in_circle=True 是 plan-runner 测试的新默认假设.

2. **alembic batch_alter_table 处理 SQLite FK nullable**: s9_2 用 `batch_alter_table + alter_column nullable=True` 让 Draft.plan_id nullable. SQLite 不支持原生 ALTER COLUMN,batch 模式自动 table rebuild.

3. **Draft.plan_id nullable 是 M4 必须的**: thesis_breach SELL drafts 由系统生成,不属于任何 plan. 否则要用 "system" 假 plan,语义混淆.
